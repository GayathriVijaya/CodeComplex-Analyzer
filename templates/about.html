What is Cyclomatic Complexity?

Cyclomatic Complexity is a software metric that measures the complexity of a program by counting the number of linearly independent paths through the source code.

  • Measures code maintainability and readability
  • Estimates testing effort required
  • Identifies risk of defects and bugs
  • Provides overall code quality assessment
  • Helps in refactoring decisions

Lower complexity values indicate simpler, more maintainable code, while higher values suggest complex logic that may be difficult to test and maintain.

Thomas J. McCabe
Creator of Cyclomatic Complexity

Complexity Calculation Formula

The cyclomatic complexity is calculated using a simple formula based on decision points in your code:

M = D + 1

Where M is the cyclomatic complexity and D is the number of decision points in the code.

Decision Points Include:

If Statements

Each if, else if condition

Loops

While, for, do-while loops

Switch Cases

Each case statement

Conditional Operators

Ternary operators (?:)

Logical Operators

&&, || in conditions

Exception Handling

Try-catch blocks
